# sort

## merge sort

分治/递归算法 整体 => 部分 => 整体

1. 将 n 个元素数组 二分
2. 递归分解
3. reduce 归并多个拆分的数组(已经排好序)为一个数组

复杂度  
O(NlogN)

## quick sort

分治思路 整体 => 部分 对数组如下操作

1. 取 start 数字为 base
2. 将小于 base 的放在左边, 大于 base 的放在右边
3. 得出 base 应该放置的位置
4. 分别对左边和右边再次进行前三步

复杂度
平均情况 O(nlogn)

- 并且常量 C 比归并排序小
- 两边能够比较均匀的均分 C 接近 1
- 每一次分的时间为 n, 常量在变小快

最糟情况 O(n^2)

- 单边一直为空调用栈为 n, 类似于冒泡排序
- 每次切换的时候为 n, 常量变小慢
