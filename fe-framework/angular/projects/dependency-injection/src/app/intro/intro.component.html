For instance, we may want to:

  1. substitute out the implementation of B for MockB during testing
  2. share a single instance of the B class across our whole app (e.g. the Singleton pattern)
  3. create a new instance of the B class every time it is used (e.g. the Factory pattern)
  4. Dependency Injection can solve these problems.
    Dependency Injection (DI) is a system to make parts of our program accessible to other parts of the program - and we can configure how that happens.

概念

  1. 在 Angular 中注册服务 , 组件中注入服务 (组件不用关系如何创建 dependencies, 只需要如何使用他，即可)
  2. Provider(提供商)：绘制token(class or string) 普通的类或者字符串 到依赖列表中，告诉Angular 怎么创建object
  3. Injector(注入器): 创建object
  4. Dependency: 被注入的对象
